# Shopee real action audit

Cập nhật: 2026-05-15.

Mục tiêu của audit này là tách rõ dữ liệu cache/preview/local với request thật lên Shopee Open Platform. Không được ghi “đã xong”, “ghi có verify”, “đã gửi sàn” nếu chưa có Shopee response và refetch verify.

## Kết luận nhanh

- Root cause chính: UI khuyến mãi đang mô tả quá mức. Nhiều card ghi “đã xong”, “ghi có verify”, “có endpoint apply” trong khi phần lớn luồng chỉ sync read-only, dựng preview, lưu queue hoặc cache D1.
- Shopee Ads phải dùng `ads_client` và biến `SHOPEE_ADS_*`. App `ADS GIADUNGHUYVAN` có App Category `Ads Service`, không được dùng để mặc định gọi Discount/Voucher/Bundle/Add-On/Flash Sale.
- Product, model, stock, price, Discount, Voucher, Bundle Deal, Add-On Deal và Shop Flash Sale phải dùng `marketplace_client` với `SHOPEE_MARKETPLACE_*` hoặc shop row đã authorize app phù hợp.
- Live write hiện bị khóa bởi `SHOPEE_LIVE_WRITE_ENABLED=false`. Khi false, backend trả `live_write_disabled`, `sent_to_shopee=false`, không fake success.
- Diagnostics mới: `POST /api/admin/shopee/diagnostics`, frontend `apps/fe/pages/admin-shopee-diagnostics.html`. Route này chỉ gọi endpoint đọc để kiểm app/token/quyền/error thật.

## Diagnostics cần chạy sau deploy

1. Mở `https://shophuyvan-analytics.nghiemchihuy.workers.dev/pages/admin-shopee-diagnostics.html` bằng profile admin production.
2. Chạy diagnostics với shop cần kiểm.
3. Đối chiếu từng `request_id` trong Shopee Console > API Access Log.
4. Nếu Ads pass nhưng Marketplace fail, không dùng app Ads Service để gọi Discount/Voucher/Bundle/Add-On/Flash Sale.
5. Chỉ bật `SHOPEE_LIVE_WRITE_ENABLED=true` sau khi diagnostics pass và người dùng xác nhận thao tác test nhỏ.
