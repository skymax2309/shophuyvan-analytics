Shop có API sẽ được preview payload trước rồi mới cho đẩy thật. Shop không có API chỉ hiện hướng dẫn fallback để tránh gửi nhầm request quảng cáo.

API thật: đang tảiShop tham chiếu: đang tảiShop đang chọn: chưa rõLog: đang tải

Dùng tab này để kiểm ADS chuẩn nhất: lấy snapshot mới trước, kiểm shop có API hay không, preview payload, rồi mới đẩy thật nếu endpoint và confirm đều đúng.

1
Kéo dữ liệu ADS trước khi quyết định

Chọn đúng khoảng ngày ở thanh trên cùng, bấm Kéo ADS theo khoảng lọc hoặc Kéo ADS tháng đã chọn. Dashboard chỉ đọc snapshot Ads API đã lưu, không lấy cost setting để đo hiệu quả.

2
Chọn đúng shop và nhánh vận hành

Mở tab Shop / API. Shop có API mới có thể preview/apply qua endpoint. Shop không API chỉ dùng để đối chiếu và thao tác tay trên Seller Center.

3
Nhập ID từ dữ liệu Ads API, không đoán

Dùng campaign/adgroup ID lấy từ bảng SKU/campaign hoặc từ sàn. Chọn đúng phạm vi, route ghi và action; field nào không liên quan sẽ tự ẩn để tránh nhập nhầm.

4
Preview trước, đọc payload và log rồi mới apply

Bấm Preview guard để xem endpoint, request_id, payload và điều kiện cho apply. Chỉ nhập câu xác nhận và bấm Đẩy thật khi payload đúng với việc cần làm.

  • Không tắt/bật campaign dựa trên dữ liệu cũ hoặc shop chưa API.
  • Nếu preview báo thiếu token/quyền/endpoint, dừng tại đó và xử lý quyền API trước.
  • Sau khi apply, mở tab Log đối soát để kiểm request_id, trạng thái và payload đã gửi.
Guard ADS đang chạm trực tiếp tiền quảng cáo. Mọi thao tác ghi thật phải đi qua preview, quyền shop API, câu xác nhận và log.
